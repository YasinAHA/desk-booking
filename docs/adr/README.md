# ADR Index

Architecture Decision Records (ADR) del proyecto.

## Uso
- Crear ADR cuando una decisión tenga impacto técnico duradero.
- Referenciar cada ADR desde `docs/DECISIONS.md`.

## Convención
- `ADR-0001-<titulo>.md`
- `ADR-0002-<titulo>.md`
- Numeracion unica para todo el monorepo.
- Cada ADR debe incluir:
  - `Status`
  - `Date`
  - `Scope` (`core`, `backend`, `frontend`)
  - `Context`
  - `Decision`
  - `Consequences`
